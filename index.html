📖 Documentation — Outil Complet Atelier

Chiffrage Financier • Gestion de Stock • Valorisation FIFO • Prestations

1. Présentation générale

Outil Complet Atelier - La Pallanterie est une application web monopage (SPA) autonome conçue pour la gestion financière et opérationnelle d'un atelier de travail adapté.

L'outil permet de :

  • Chiffrer des prestations de services (coût de revient, prix de vente, marges)
  • Gérer les stocks avec valorisation FIFO automatique
  • Suivre les clients et leur historique de prestations
  • Planifier les prestations via un diagramme de Gantt
  • Analyser la rentabilité et les performances via des tableaux de bord

L'application fonctionne entièrement hors ligne dans le navigateur, sans serveur, sans installation. Toutes les données sont stockées localement via le localStorage du navigateur.

2. Démarrage rapide

Installation : aucune. Ouvrez simplement index.html dans un navigateur moderne (Chrome, Firefox, Edge).

Premier lancement :

  1. Accédez à l'onglet Paramétrage : coût horaire CEA/MSP, frais généraux, taux de marge par défaut
  2. Importez votre catalogue de gestes via Gestes → Import CSV
  3. Créez votre première prestation dans le Calculateur

Sauvegarde initiale : dès que des données sont saisies, exportez via Configuration → Export JSON.

4. Modules fonctionnels (14 onglets)

L'interface est organisée en 14 onglets accessibles via la barre de navigation. Selon le profil actif, certains onglets peuvent être masqués.

4.1 🏠 Tableau de bord

  • Devis en attente (alertes et actions rapides)
  • Pipeline de chiffre d'affaires sur 3 mois
  • Top clients par CA sur 12 mois
  • Indicateurs d'alerte (stock, devis expirés)
  • Cadres théoriques recommandés

4.2 🧙 Assistant (admin)

  • Assistant pas-à-pas pour construire une prestation
  • Modèles de services pré-configurés
  • Analyse de situations et recommandations théoriques
  • Support à la formation des encadrants

4.3 💼 Calculateur — module central

Informations de la prestation : client, intitulé, référence, date, responsable.

Personnel : CEA (travailleurs) et MSP (encadrants).

Gestion des gestes :

  • Ajouter depuis le catalogue ou créer à la volée
  • Durée (minutes) et quantité par pièce
  • Réorganiser par glisser-déposer, favoris, annuler/refaire (Ctrl+Z)
  • Bundles : raccourcis de combinaisons de gestes
  • Chronomètre intégré pour mesurer le temps réel

Valorisation financière : coût MO CEA + MSP, consommables, stockage, frais généraux, coût de revient, marge, prix de vente.

Simulation de volumes : coût unitaire pour 100, 250, 500, 1 000 et 2 500 unités.

Export PDF : version client (sans coûts) ou version interne (détail complet).

4.4 📦 Gestion de stock

  • Mouvements : entrées, sorties, transferts avec traçabilité complète
  • Stock actuel en temps réel (quantité, valeur FIFO, emplacement)
  • Valorisation FIFO automatique (Premier Entré, Premier Sorti)
  • Alertes automatiques sous le seuil minimum défini

5. Calculs et méthodes financières

Coût de revient = Coût MO CEA + Coût MO MSP + Consommables + Stockage + Frais généraux
Coût MO       = (Temps en h) × Nb agents × Taux horaire chargé
Frais stockage = Nb jours × Taux journalier
Prix de vente  = Coût de revient × (1 + Taux marge / 100)
Valeur sortie  = Σ (Quantité lot × Prix unitaire lot)  [FIFO]

Le temps total intègre la somme des gestes × coefficient × quantité par pièce × nombre de pièces, plus le temps administratif additionnel. La simulation de volumes identifie les seuils de rentabilité pour 100, 250, 500, 1 000 et 2 500 unités.

6. Import / Export de données

Export JSON (Config → Export JSON) : sauvegarde complète — gestes, consommables, prestations, clients, stock, mouvements, paramètres.

Import JSON (Config → Import JSON) : restauration depuis une sauvegarde. Attention : remplace les données existantes.

Import CSV gestes (Gestes → Import CSV) : format code, categorie, description, duree_minutes, coefficient, notes.

Export PDF (Calculateur ou Clients) : version client (prix de vente uniquement) ou version interne (détail complet des coûts et marges).

10. Données et persistance

Toutes les données sont enregistrées dans le localStorage du navigateur (JSON). Elles persistent entre les sessions tant que le cache n'est pas effacé.

Recommandation : export JSON hebdomadaire minimum. Le localStorage est limité à ~5–10 Mo selon le navigateur.

Situations cliniques en atelier

📚 Référentiel Théorique

Cliquez sur un thème pour développer les ressources et liens de référence.

🧠 Piaget — Développement cognitif & Zone proximale▶

Concepts clés : Assimilation, Accommodation, Déséquilibre cognitif, Zone proximale de développement (ZPD), Stades opératoires.

En atelier : Une personne en déséquilibre cognitif montre des erreurs répétées ou un ralentissement face à une tâche dépassant sa ZPD. La posture MSP = séquencer, faire avec, puis laisser faire.

Concepts clés : Double bind (double contrainte), Injonctions contradictoires non verbalisables, Contexte d'apprentissage pathogène, Deutero-apprentissage.

En atelier : "Sois autonome" + "chaque erreur est sanctionnée" = double contrainte. La personne est paralysée. Posture MSP = nommer explicitement la contradiction et lever l'injonction impossible.

En atelier : Conflits territoriaux, immixtion du MSP dans la relation entre pairs, coalitions = indicateurs de frontières mal définies. Posture MSP = clarifier les rôles, les espaces et les responsabilités.
